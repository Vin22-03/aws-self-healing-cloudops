Self-Healing CloudOps on AWS

A CloudOps-focused project demonstrating automated failure detection, remediation, and recovery using AWS managed services.

Objective

  • Detect ECS service failures automatically
  • Trigger remediation without human intervention
  • Reduce MTTR using event-driven automation

Solution Overview

  • CloudWatch alarms for detection
  • EventBridge for incident routing
  • Lambda for automated remediation
  • SNS for operator notifications

Failure Scenarios

  • Health check failure
  • ECS task crash
  • Service instability

CloudOps Philosophy

  • Detect early
  • Automate remediation
  • Humans are informed, not required
